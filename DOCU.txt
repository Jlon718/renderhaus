1. INTRODUCTION
================================================================================

1.1 PROJECT OVERVIEW

RenderHaus is a web-based 3D interior design platform that enables users to 
create, visualize, and customize interior spaces using three-dimensional 
furniture models. The system integrates artificial intelligence to provide 
intelligent color and furniture recommendations, helping users make informed 
design decisions. Built with modern web technologies, RenderHaus offers an 
intuitive interface for both novice and experienced designers to bring their 
interior design visions to life.

1.2 PURPOSE

The primary purpose of RenderHaus is to:
- Democratize interior design by making 3D visualization accessible to everyone
- Reduce the cost and time associated with traditional interior design planning
- Provide AI-powered recommendations to assist in design decision-making
- Enable users to experiment with different furniture arrangements and colors
  before making purchasing decisions
- Generate professional design exports suitable for client presentations

1.3 SCOPE

RenderHaus provides the following capabilities:
- User authentication and account management
- 3D furniture model library with categorized browsing
- Interactive 3D canvas for furniture placement and arrangement
- Real-time object manipulation (position, rotation, scale, color)
- AI-powered color recommendations based on furniture type and style
- AI-powered furniture suggestions based on room analysis
- Project management (save, load, delete multiple designs)
- PDF export functionality with AI recommendations
- Administrative tools for model and user management

1.4 TARGET USERS

Primary Users:
- Homeowners planning room renovations or new furniture purchases
- Interior design students learning spatial arrangement
- Small business owners designing office or retail spaces
- Real estate agents staging properties virtually

Administrative Users:
- System administrators managing 3D model library
- Content managers uploading and maintaining furniture models
- Platform administrators monitoring user activity

1.5 SYSTEM ARCHITECTURE

RenderHaus employs a three-tier microservices architecture:

Frontend (React + Three.js):
- User interface and 3D visualization
- Runs on port 3000
- Communicates with backends via RESTful APIs

Backend 1 (Node.js + Express):
- Authentication and authorization
- Database operations (CRUD)
- File upload management
- Runs on port 5000

Backend 2 (Python + Flask):
- AI color recommendation engine
- Furniture suggestion algorithms
- 3D model analysis
- Runs on port 5001

Database (MongoDB Atlas):
- Cloud-hosted NoSQL database
- Stores users, models, projects, templates

Storage (Cloudinary):
- Cloud storage for 3D models and images
- CDN for fast global content delivery

1.6 TECHNOLOGY STACK

Frontend:
- React 19.1.0 - UI framework
- Three.js 0.178.0 - 3D rendering engine
- @react-three/fiber - React integration for Three.js
- @react-three/drei - 3D helpers and controls
- jsPDF - PDF generation
- html2canvas - Canvas screenshot capture

Backend (Node.js):
- Express.js - Web framework
- Mongoose - MongoDB ODM
- JWT - Authentication
- Bcrypt - Password hashing
- Cloudinary - File storage
- Multer - File upload handling
- Nodemailer - Email service

Backend (Python):
- Flask - Web framework
- PyMongo - MongoDB driver
- NumPy - Numerical computing
- scikit-learn - ML library foundation
- Trimesh - 3D mesh processing
- Pillow - Image processing

Database:
- MongoDB Atlas - Cloud database

1.7 KEY FEATURES

3D Visualization:
- WebGL-based 3D rendering without plugins
- Support for GLTF/GLB model formats
- Real-time camera controls (orbit, pan, zoom)

AI Intelligence:
