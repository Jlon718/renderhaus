=== FRONTEND (React Application) ===

CORE FRAMEWORK:
- React 18.x - JavaScript library for building user interfaces

3. SYSTEM FLOW & USER JOURNEY
================================================================================

=== REGISTRATION & LOGIN ===

STEP 1: User Registration
File: frontend/src/components/Register.js
Flow:
1. User enters email, password, username
2. Frontend validates input format
3. POST /api/auth/register ‚Üí backend/routes/auth.js
4. authController.register() hashes password with bcryptjs
5. Creates User document in MongoDB
6. Sends OTP via Nodemailer to user's email
7. User enters OTP for verification
8. Account activated

STEP 2: User Login
File: frontend/src/components/Login.js
Flow:
1. User enters email & password
2. POST /api/auth/login ‚Üí backend/controllers/authController.js
3. Compare password hash using bcryptjs
4. Generate JWT tokens (access + refresh)
5. Return tokens to frontend
6. Store tokens in localStorage
7. Redirect to Dashboard

=== MAIN DESIGN WORKFLOW ===

STEP 3: Dashboard & 3D Canvas
File: frontend/src/components/Dashboard.js
Components:
- Sidebar (left) - Navigation between sections
- Canvas (center) - 3D scene rendering
- Placed Models List (right-center) - Models currently in scene
- Properties Panel (right) - Edit selected object

STEP 4: Browse & Add Furniture
File: frontend/src/components/ModelLibrary.js
Flow:
1. Click "Model Library" in sidebar
2. GET /api/models ‚Üí backend/controllers/modelController.js
3. Fetch all 3D models from MongoDB
4. Display thumbnails with categories
5. User clicks model thumbnail
6. Model added to placedModels array in Dashboard state
7. Canvas.js receives updated placedModels
8. Model3D.js component renders the 3D model using Three.js

STEP 5: 3D Rendering
File: frontend/src/components/Canvas.js
Technologies: @react-three/fiber, @react-three/drei, Three.js
Flow:
1. Canvas component creates Three.js scene
2. For each model in placedModels:
   - Model3D component loads .glb file using GLTFLoader
   - Apply position, rotation, scale transformations
   - Render mesh with materials
3. OrbitControls enables camera rotation/zoom
4. Raycasting detects object clicks for selection

STEP 6: Edit Object Properties
File: frontend/src/components/Functions/ObjectProperties.js
Features:
- Position controls (X, Y, Z axes with sliders)
- Scale control (0.1x to 3x size)
- Rotation control (0¬∞ to 360¬∞)
- Color picker
- AI color recommendations

STEP 7: AI Color Recommendations
Files: 
- Frontend: frontend/src/components/Functions/ColorPicker.js
- Backend: backend/routes/aiColorSuggestions.js
Flow:
1. User selects object in 3D scene
2. Click "üé® AI Color Suggestions" button
3. POST /api/python/get-ai-color-suggestions
4. Backend analyzes:
   - Object name (e.g., "Modern Sofa")
   - Object category (e.g., "furniture-seating")
   - Current color
5. detectObjectType() function uses regex to categorize
6. detectStyleFromName() extracts style (modern/traditional/rustic)
7. Returns 4 contextual colors from color schemes
8. Display color swatches with descriptions
9. User clicks color to apply

STEP 8: AI Furniture Suggestions
Files:
- Frontend: frontend/src/components/Functions/AISuggestSection.js
- Backend: backend/python_backend/ai_suggestions.py
Flow:
1. User clicks "AI Suggestions" in sidebar
2. POST /api/python/ai/suggestions with current furniture
3. Python backend analyzes:
   - Count furniture by category
   - Identify missing essentials (seating, tables, lighting)
   - Detect room style from model names
4. Generate suggestions for missing items
5. Query MongoDB for actual models matching suggestions
6. Return generic suggestions + specific model recommendations
7. Display furniture and color suggestions

STEP 9: Export to PDF
File: frontend/src/components/Functions/ExportSection.js
Flow:
1. User clicks "Export to PDF" button
2. Capture 3D canvas using html2canvas
3. Fetch latest AI suggestions
4. Generate PDF using jsPDF with:
   - Project metadata (date, timestamp)
   - Table of placed models
   - 3D design screenshot
   - AI furniture suggestions
   - Recommended models from database
   - AI color palette
5. Save PDF file to user's computer

STEP 10: Save Project
File: frontend/src/components/ProjectManager.js
Flow:
1. User clicks "Save Project"
2. POST /api/projects with:
   - Project name
   - Placed models array (positions, colors, transforms)
   - Thumbnail (canvas screenshot)
3. Create Project document in MongoDB
4. Link to current user's account

STEP 11: Load Existing Project
Flow:
1. GET /api/projects ‚Üí fetch user's projects
2. User selects project from list
3. Load placedModels array into Dashboard state
4. Canvas re-renders with saved furniture
5. Restore all positions, colors, transformations

=== OVERVIEW ===
The AI color system is RULE-BASED, not machine learning.
It uses pattern matching and predefined color schemes.

=== KEY FILE TO SHOW ===
üìÅ File: backend/routes/aiColorSuggestions.js
Lines: 1-311 (entire file)

=== HOW IT WORKS (Step-by-Step) ===

STEP 1: Receive Request
Frontend sends:
{
  "objectName": "Modern Sofa",
  "objectCategory": "furniture-seating",
  "currentColor": "#8B4513",
  "context": "interior_design"
}

STEP 2: Detect Object Type
Function: detectObjectType(objectName, objectCategory)
Location: Lines 156-202

Algorithm:
1. Combine name and category into searchable text
2. Use regex patterns to match object types:
   - /sofa|couch|chair|armchair|bench|stool|ottoman/i ‚Üí "seating"
   - /table|desk|counter/i ‚Üí "tables"
   - /lamp|light|chandelier|sconce/i ‚Üí "lighting"
   - /cabinet|shelf|wardrobe|dresser|storage/i ‚Üí "storage"
   - /bed|mattress|headboard/i ‚Üí "beds"
   - /curtain|rug|carpet|pillow|cushion/i ‚Üí "textiles"
   - /vase|frame|sculpture|plant|decor/i ‚Üí "decoration"
   - /kitchen|appliance|sink|stove/i ‚Üí "kitchen"
3. Return matched type or "general" as fallback

Example:
Input: "Modern Sofa", "furniture-seating"
Regex match: /sofa/ matches ‚Üí return "seating"

STEP 3: Detect Style
Function: detectStyleFromName(objectName)
Location: Lines 204-222

Algorithm:
1. Check object name for style keywords:
   - Contains "modern", "contemporary", "minimalist" ‚Üí "modern"
   - Contains "traditional", "classic", "vintage", "antique" ‚Üí "traditional"
   - Contains "rustic", "farmhouse", "industrial", "reclaimed" ‚Üí "rustic"
2. Default to "modern" if no match

Example:
Input: "Modern Sofa"
Match: "modern" found ‚Üí return "modern"

STEP 4: Select Color Scheme
Function: generateColorSuggestions()
Location: Lines 35-154

Data Structure:
colorSchemes = {
  seating: {
    modern: [
      { name: 'Charcoal Gray', hex: '#36454F', description: '...' },
      { name: 'Navy Blue', hex: '#1E3A8A', description: '...' },
      { name: 'Soft Beige', hex: '#D4C5B9', description: '...' },
      { name: 'Olive Green', hex: '#4A5D23', description: '...' }
    ],
    traditional: [...],
    rustic: [...]
  },
  tables: {...},
  lighting: {...},
  // ... 8 categories total
}

Logic:
1. Get color array: colorSchemes[objectType][style]
2. If not found, use colorSchemes["general"][style]
3. Return all 4 colors in that scheme

Example:
- objectType = "seating"
- style = "modern"
- Returns: 4 colors from colorSchemes.seating.modern

STEP 5: Return Results
Response:
{
  "status": "success",
  "colors": [
    { "name": "Charcoal Gray", "hex": "#36454F", "description": "..." },
    { "name": "Navy Blue", "hex": "#1E3A8A", "description": "..." },

=== COLOR SCHEMES EXPLANATION ===

Why These Colors?
- Based on interior design principles
- 3 style variations per object type (9 styles √ó 8 types = 72 schemes)
- Each scheme has 4 complementary colors
- Colors chosen for:
  * Furniture type appropriateness
  * Style consistency
  * Interior design trends
  * Color harmony

Example Reasoning:
SEATING (Sofas/Chairs):
- Modern: Gray/Navy/Beige (neutral, professional)
- Traditional: Burgundy/Brown/Cream (warm, classic)
- Rustic: Earth tones/Wood tones (natural)

LIGHTING:
- Modern: Black/Silver/Gold (metallic, sleek)
- Traditional: Brass/Bronze/Copper (warm metals)
- Rustic: Iron/Copper/Black (industrial)

=== WHY NOT MACHINE LEARNING? ===

Current Approach: Rule-Based System
Pros:
‚úì Fast and predictable
‚úì No training data needed
‚úì Consistent results
‚úì Easy to debug and explain
‚úì Works immediately without dataset

Future ML Enhancement:
- Collect user color choices
- Train model on preference patterns
- Consider room context (size, lighting, existing colors)
- Use computer vision to analyze uploaded images
- Recommend colors based on successful designs

Libraries for Future ML:
- TensorFlow.js (JavaScript ML)
- scikit-learn (already installed, Python)
- OpenCV (computer vision)

=== CANVAS.JS EXPLAINED ===

Key Sections:

1. Scene Setup
```javascript
<Canvas
  camera={{ position: [5, 5, 5], fov: 50 }}
  style={{ background: '#f0f0f0' }}
>
```
- Creates WebGL context
- Sets camera position (x:5, y:5, z:5)
- Field of view: 50 degrees

2. Lighting
```javascript
<ambientLight intensity={0.5} />
<directionalLight position={[10, 10, 5]} intensity={1} />
```
- Ambient: Uniform lighting (0.5 intensity)
- Directional: Simulates sunlight from position [10,10,5]

3. Camera Controls
```javascript
<OrbitControls
  enablePan={true}
  enableZoom={true}
  enableRotate={true}
/>
```
- Mouse drag: Rotate camera
- Scroll: Zoom in/out
- Right-click drag: Pan camera

4. Grid Helper
```javascript
<gridHelper args={[20, 20]} />
```
- Creates floor grid (20√ó20 units, 20 divisions)
- Helps visualize space and scale

5. Model Rendering
```javascript
{placedModels.map((model) => (
  <Model3D
    key={model.id}
    modelData={model}
    isSelected={selectedObject?.id === model.id}
    onClick={() => handleModelClick(model)}
  />
))}
```
- Loops through all placed models
- Renders each with Model3D component
- Handles selection highlighting

=== MODEL3D.JS EXPLAINED ===

Key Sections:

1. Load 3D Model
```javascript
const gltf = useLoader(GLTFLoader, modelData.fileUrl);
```
- GLTFLoader: Three.js loader for .glb/.gltf files
- useLoader: React Three Fiber hook for async loading
- Returns: GLTF object with scene, meshes, materials

2. Apply Transformations
```javascript
<group
  position={modelData.position || [0, 0, 0]}
  rotation={modelData.rotation || [0, 0, 0]}
  scale={modelData.scale || [1, 1, 1]}
  onClick={onClick}
>
  <primitive object={gltf.scene} />
</group>
```
- group: Container for transformations
- position: [x, y, z] coordinates
- rotation: [x, y, z] in radians
- scale: [x, y, z] multiplier
- primitive: Renders the loaded 3D model

3. Color Override
```javascript
useEffect(() => {
  if (modelData.color && gltf) {
    gltf.scene.traverse((child) => {
      if (child.isMesh) {
        child.material = new THREE.MeshStandardMaterial({
          color: modelData.color
        });
      }
    });
  }
}, [modelData.color, gltf]);
```
- Traverse: Loop through all meshes in model
- Check: Is it a mesh? (not light, camera, etc.)
- Replace: Material with new color
- MeshStandardMaterial: PBR (Physically Based Rendering)

4. Selection Highlighting
```javascript
if (isSelected) {
  // Add bounding box or outline
  const box = new THREE.BoxHelper(gltf.scene, 0xffff00);
  scene.add(box);
}
```
- Yellow bounding box around selected object

=== 3D FILE FORMAT ===

Format: GLTF/GLB
- GLTF: JSON-based, human-readable
- GLB: Binary, smaller file size
- Industry standard for web 3D
- Supports: Meshes, materials, textures, animations

File Structure:
.glb file contains:
- Geometry (vertices, faces)
- Materials (colors, textures)
- Textures (image files embedded)
- Scene hierarchy
- Metadata

=== PERFORMANCE OPTIMIZATIONS ===

1. Lazy Loading
- Models load only when placed in scene
- React Suspense fallback during loading

2. Level of Detail (LOD)
- Could implement: Load low-poly versions first
- Swap to high-poly when camera is close

3. Instancing
- Reuse same geometry for duplicate objects
- Saves memory for repeated furniture

4. Culling
- Don't render objects outside camera view
- Three.js does this automatically

Q2: "How does the 3D rendering work?"

A: "We use Three.js, a WebGL-based 3D graphics library, integrated with 
React through @react-three/fiber.

TECHNICAL FLOW:
1. GLTFLoader loads .glb files (binary 3D models)
2. Three.js creates a Scene, Camera, and Renderer
3. OrbitControls provides camera manipulation
4. Each furniture model is a Group with:
   - Position (x, y, z coordinates)
   - Rotation (euler angles)
   - Scale (size multiplier)
5. Raycasting detects mouse clicks on 3D objects
6. Materials are PBR (Physically Based Rendering) for realistic lighting

Show: frontend/src/components/Canvas.js and Model3D.js

WHY WEBGL:
- Runs on GPU, hardware-accelerated
- No plugins required (native browser support)
- Cross-platform (Windows, Mac, mobile)
- Industry standard for web 3D"

Show: backend/routes/aiColorSuggestions.js

INTELLIGENCE FACTORS:
1. OBJECT TYPE DETECTION (Lines 156-202)
   - Analyzes object name and category
   - Uses regex patterns to classify into 8 categories
   - Example: 'sofa|couch|chair' ‚Üí 'seating'

Q5: "What security measures are implemented?"

A: "Multiple security layers:

AUTHENTICATION:
- JWT (JSON Web Tokens) for stateless authentication
- Access tokens expire in 15 minutes
- Refresh tokens for 7 days
- Tokens stored in localStorage (HTTPS only in production)

PASSWORD SECURITY:
- Bcrypt hashing (10 salt rounds)
- Never store plain text passwords
- Hashes are one-way (cannot be reversed)

EMAIL VERIFICATION:
- OTP (One-Time Password) sent via email
- Prevents fake account creation
- OTP expires after 10 minutes

INPUT VALIDATION:
- express-validator on all inputs
- Prevents SQL injection (using MongoDB ODM)
- XSS protection (sanitize user inputs)

AUTHORIZATION:
- Middleware checks JWT on protected routes
- Role-based access control (user vs admin)
- Users can only access their own projects

CORS:
- Configured to only allow frontend domain
- Prevents unauthorized API access

FILE UPLOAD:
- Multer limits file size (10MB)
- File type validation (only .glb, .gltf)
- Cloudinary scans for malware

ENVIRONMENT VARIABLES:
- Sensitive data in .env (not in git)
- JWT secret, database credentials hidden"

LOADING STRATEGY:

- React Suspense with loading fallback

- Cache loaded models in browser memory

FUTURE IMPROVEMENTS:
